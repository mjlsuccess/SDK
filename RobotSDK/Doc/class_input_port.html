Detailed Description

Class InputPort is the input port for node class to receive input data.

  • Input data is wrapped in the form of Qt signal generated by OutputPort.
  • InputPort is an universal port to receive this signal.
  • It is the front-end of Node.
    • Source: 0 port.
    • Drain: >0 port(s).
    • Processor: >0 port(s).
    • SourceDrain: >0 port(s).
  • It contains the previous node's parameters associated with the latest input data.
  • It contains a data buffer InputPort::inputdatabuffer to store the most recent input data in the size of InputPort::inputbuffersize.
  • It will run in an independent thread Node::inputportsthread to guarantee the data-stream will not be blocked.

InputPort::InputPort ( int  inputBufferSize)

Determines the size of InputPort::inputdatabuffer.

Parameters
[in]inputBufferSizeThe size of the InputPort::inputdatabuffer.
  • If inputBufferSize>0, the length of the databuffer is limited to inputBufferSize.
  • If inputBufferSize<=0, the length of the databuffer is unlimited, which means all input data will be stored. It is strongly recommend not to set grabSize>0 in this mode.
  • The function InputPort::grabInputData(int grabSize) will grab and remove the stored data.

QVector< boost::shared_ptr< void > > InputPort::grabInputData ( int  grabSize)

Grab the input data in the size of grabSize from InputPort::inputdatabuffer.

Parameters
[in]grabSizeThe size of grabbed input data.
Returns
The grabbed input data in QVector.
  • grabSize
    • grabSize==0 : all input data are grabbed. If InputPort::inputbuffersize<=0, all input data are removed from InputPort::inputdatabuffer.
    • grabSize>0 : grab the most recent input data in the size of grabSize without any removement. It is strongly recommend not to set InputPort::inputbuffersize=0 in this mode.
    • grabSize<0 : grab the most ancient input data of InputPort::inputdatabuffer in the size of grabSize. If InputPort::inputbuffersize<=0, the grabbed input data are removed from InputPort::inputdatabuffer.
  • The return value
    • Front-end is the latest grabbed data
    • Back-end is the oldest grabbed data
